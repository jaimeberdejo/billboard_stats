# Requirements

**Milestone:** v1.0 Initial Next.js Release
**Status:** SHIPPED — all 19 requirements validated.

**Outcomes:** Every v1.0 requirement (CORE / BROWSE / DETAILS / SEARCH / RECORDS) was
delivered and verified. CORE-02 (Neon connectivity) and CORE-03 (Vercel deploy) were
scoped to Phase 1 but fully realized in Phase 5 when production Neon + Vercel went live.
Phase 6 (added post-roadmap) backfilled production data and automated the weekly ETL —
not tied to a v1.0 requirement ID but necessary to keep BROWSE-05 freshness accurate.

## Requirements (v1.0) — all validated

### CORE: Infrastructure & Navigation

- [x] **CORE-01**: Setup Next.js App Router project with TypeScript and Tailwind
- [x] **CORE-02**: Connect to Neon PostgreSQL database and setup typed API routes replacing Python services
- [x] **CORE-03**: Deploy application to Vercel
- [x] **CORE-04**: Implement mobile-responsive layout (sticky top nav desktop, bottom nav mobile)
- [x] **CORE-05**: Implement tabular-nums font-variant and dense data-first design (Space Grotesk)

### BROWSE: Latest Charts

- [x] **BROWSE-01**: Display Latest Charts (Hot 100 & B200) in ranked tables with sticky headers
- [x] **BROWSE-02**: Toggle between Hot 100 and Billboard 200
- [x] **BROWSE-03**: Select specific historical chart weeks via dropdown
- [x] **BROWSE-04**: Display movement indicators (green up, red down, gray flat, NEW/RE badges)
- [x] **BROWSE-05**: Display data status / freshness indicator (row counts and latest chart dates)

### DETAILS: Entity Pages

- [x] **DETAILS-01**: Song detail page with stats bar (peak, total weeks, debut, etc) and artist pill links
- [x] **DETAILS-02**: Album detail page with stats bar and artist pill links
- [x] **DETAILS-03**: Artist detail page with career aggregates (max simultaneous songs) and Hot 100/B200 tables
- [x] **DETAILS-04**: Week-by-week chart history table for songs and albums
- [x] **DETAILS-05**: Chart run SVG visualization (inline, collapsible, y-axis inverted, peak annotated)

### SEARCH: Discoverability

- [x] **SEARCH-01**: Fuzzy search across songs, albums, and artists (min 2 chars)
- [x] **SEARCH-02**: Tabbed search results (Songs/Albums/Artists) showing exact result counts

### RECORDS: Leaderboards & Queries

- [x] **RECORDS-01**: Preset leaderboards (most weeks at #1, longest runs, biggest debuts, etc)
- [x] **RECORDS-02**: Custom Query Builder filtering by metric, sort, artist, peak, debut, and min-weeks
