<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell → vercel env add | Neon pooled connection string (with password) transmitted to Vercel API |
| .env.local on disk | Contains DATABASE_URL with credentials; must be gitignored |
| Vercel build environment | DATABASE_URL injected at build time as encrypted secret |
| Public production URL | App is public — all routes are read-only, no auth surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Information Disclosure | DATABASE_URL in shell history (vercel env add) | mitigate | Plan instructions note risk and recommend interactive prompt (not piping) so password is not visible in history. Vercel stores env vars encrypted at rest. |
| T-05-06 | Information Disclosure | .env.local on local filesystem | mitigate | `.gitignore` already has `.env*` pattern (verified in .gitignore). `vercel env pull` writes to .env.local which is excluded. Task 2 acceptance criteria verify gitignore works. |
| T-05-07 | Tampering | Push to main triggers auto-deploy | accept | This is intentional by D-06. GitHub authentication protects the repo. Only authorized committers can trigger auto-deploy. |
| T-05-08 | Information Disclosure | Vercel project.json committed to git | accept | `.vercel/project.json` contains only orgId and projectId (not secrets). It is NOT in .gitignore intentionally — allows teammates to link the same project. Acceptable for a public solo project. |
| T-05-09 | Elevation of Privilege | DATABASE_URL gives full DB read access via pooled connection | accept | The Neon DB is read-only from the app's perspective (no write routes). Data is public Billboard chart data (no PII). Risk is low. |
</threat_model>
